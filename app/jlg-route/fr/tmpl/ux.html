Usability test

What is it?

Observing a user having an objective and using your site to get it done.

Purpose of the mission

  • Feed the backlog of usability requirements.
  • Improve the usability level of your website.

How this is done?

Testing on the client site in front of all the stakeholders of the project. This mission absolutely requires the presence of the technical and business decision makers of the projects or it will be less productive.

  • Recruit 2 users
  • Deciding what tasks they are going to perform

The meeting duration is 3 hours max:

  • 2 hours observing users performing tasks
  • 1 hour for the debriefing with the project stakeholders

The debriefing output is the backlog of usability requirement. This mission should be done on a regular basis.

Interface audit

What is it?

Examination of all interface layout and widgets of an interface, and applying on them a robust check-list.

Purpose of the mission

Improving the user interface of an existing application or a mockup.

How this is done?

This is done by applying the Bastien-Scapin check-list, the gestalt law, and some of the Suzan Weinschenk things that every designer should know.

Prototyping

What is it?

From a really identified user objective, (re)design a portal user interface and a set of applications screen suite.

Purpose of the mission

Deliver to the development team the right mockup of an application, validated by the users.

How this is done?

You should know who your users are, what they want and propose a better way to do it than before. See the User Modeling mission to do it in a proper way.

Identify if you need a portal, and then the list of the needed applications inside.

Then you are ready for prototyping.

  1. Describe your nominal process, then decompose it to 5 majors steps max, that can be decomposed as well as 5 substeps max and recursively.
    Validate with users.
  2. Draw a storyboard in wireframe without computer tools (only pen and paper).
    Validate with users.
  3. Copy your wireframe storyboard with a prototyping tool (ex: Axure, etc.).
    Validate with users.
  4. With a graphical designer, insert the graphical elements (charts, etc.).
    Validate with users.
  5. Then give the mockup to developer.

Important note: Do not explain how your interface work to the user, or the test is simply not valid.

You may ask two things :

  • What does he understand?
  • Where does he want to click?

No help is permitted. If the user does not understand either does not know where to click to achieve his objective, then the test fails: it is a usability bug that needs to be fixed.

User modeling

What is it?

A set of activities that allows designers to see their target : the user.

Purpose of the mission

To let all the project stakeholders understanding:

  • who the users are,
  • what their objectives are ,
  • and how they would like to achieve them.

How this is done?

This is done by doing, according the situations:

  • Personae,
  • Diary studies
  • Onsite observation
  • Interview
  • Survey
